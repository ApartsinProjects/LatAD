LDT — Latent Dynamics Twin

A frozen, self-supervised, multi-resolution encoder of the past. It augments the window-based MIIM detector (CVaDE) with history context. Companion to the framework (C1–C4).

Guiding principle. The LDT does not deal with anomalies — it only encodes the past. It never forecasts the future and never produces a score. All anomaly detection (including transition validity) lives in CVaDE, which runs on the window augmented with the LDT's context. This removes the classic failure of forecasting-based AD — where unpredictable is conflated with anomalous — because the encoder neither predicts nor judges.

1. Why: the snapshot limitation

The window/mode detector sees a snapshot: no memory of the trajectory that led here. Two anomaly classes are invisible to any snapshot (even a longer window or a convolution over it): (i) a state normal now but reached via an invalid path, and (ii) a slow drift spanning far more than one window. The fix is a learned summary of history at bounded dimension, provided to the detector as extra input.

They train separately (stable, testable in isolation) and compose by a single concatenation. First verify "$c_t$ improves detection" in isolation, then layer C1–C4.

3. Multi-resolution state

The state is not one summary but a stack of sub-states, one per time scale: $c_t=[c^{\text{short}}_t\oplus c^{\text{med}}_t\oplus c^{\text{long}}_t]$ — fast detail, medium patterns, and slow trends / long history. Clean backbone: a dilated temporal-conv stack (dilations $1,2,4,\dots$), read out at three depths (shallow = short receptive field → $c^{\text{short}}$; deep = large receptive field → $c^{\text{long}}$). Equivalent options: a clockwork/hierarchical RNN with three update rates, an SSM (S4/Mamba), or a temporal pyramid over downsampled streams.

Force each branch to specialise (else all collapse to short) by tying its reconstruction target to its scale: the short branch reconstructs fine recent detail; the long branch reconstructs the coarse trend / long-window average (Slow-Feature principle). Dilation limits each branch architecturally, and the per-scale target pulls it to its own resolution.

Bonus: anomalies become scale-attributable downstream — a fast transient perturbs $c^{\text{short}}$, a slow drift perturbs $c^{\text{long}}$ — enabling per-scale interpretation and control.

4. Objective — masked reconstruction of the past (no forecasting)

The LDT is trained purely to encode history, so its objective reconstructs the past, not the future — a multi-resolution masked / denoising autoencoder over the trajectory. Mask or downsample past steps and reconstruct them from $c_t$, per scale:

$$\mathcal L_{\text{LDT}}=\sum_{r\in\{\text{short,med,long}\}}\ \lambda_r\ \big\|\,\text{past}^{(r)}-\text{dec}_r(c^{r}_t,\tilde\tau^{\,r}_t)\,\big\|^2 .$$
  • The past is determined (unlike the future), so this is always well-posed — unpredictable normal signal is never penalised; the LDT is not asked to guess anything stochastic.
  • Reconstruction is naturally collapse-resistant (the decoder forces $c_t$ to retain information) — no contrastive negatives or EMA target required. Optional VICReg / masking (MAE-style) only sharpen the representation.

4.1 Long-memory requirements (design against short-memory bias)

  • Reconstruct at long range — the long branch must reconstruct far past (coarse trend over hundreds/thousands of steps), weighted so it cannot be ignored.
  • Multi-timescale backbone — dilated-TCN / SSM / clockwork, not a vanilla RNN.
  • Coarse targets for long scales — reconstruct long-window averages / slopes, not raw values, so $c^{\text{long}}$ encodes slow structure, not detail.

What "long-term" should mean: $c_t$ must summarise the recent sequence of regimes, their dwell times, and the slow trend — not raw history. That summary is what later makes long-range invalid patterns detectable in CVaDE.

4.2 Maturity signal instead of burn-in

No hard burn-in cutoff. Feed a per-resolution maturity signal as input so the encoder knows how much history each scale has absorbed:

$$\tilde\tau^{\,r}_t=\log\!\big(1+\min(t-t_{\text{start}},\,R_r)\big),$$

saturating at branch $r$'s receptive field $R_r$ (short matures fast, long matures slowly). Because $c_t$ then explicitly represents "start / low-maturity," and training includes many run-starts, a start-state is itself a normal, dense state — so CVaDE is automatically tolerant early without discarding any steps. Maturity gates how sure the representation is, never validity.

4.3 Data & optimisation

  • Normal only; reset state and $\tau$ at run/file boundaries; per-channel standardisation on train.
  • Adam ~1e-3, gradient clipping, long subsequence batches (for long memory).
  • Label-free model selection: state dims, receptive fields, mask ratio chosen by held-out normal reconstruction loss.

4.4 Acceptance test — measure the memory horizon

Do not assume long memory; probe it. Train linear probes to recover long-past events from $c_t$ and measure the effective memory horizon. If probes cannot recover events from far back, increase long-range reconstruction weight / change backbone. This gates the LDT before it feeds CVaDE.

5. Output and how CVaDE uses it (all detection is CVaDE's)

The only LDT output is the multi-resolution context $c_t$. CVaDE runs on $\xi_t=[\phi(W_t)\oplus c^{\text{short}}_t\oplus c^{\text{med}}_t\oplus c^{\text{long}}_t]$ (standardised so no part swamps), and does everything anomaly-related:

  • Snapshot & history anomalies — an unusual window or an unusual history context makes $\xi_t$ off-manifold under the mode/density model.
  • Invalid transitions (emergent, not a special LDT score) — a bad transition means the history $c_t$ is inconsistent with the current window; the pair $(\phi(W_t),c_t)$ is off-manifold → flagged. Distinguishing rare-but-valid from invalid transitions is handled inside CVaDE by C2 (basin) and C4 (per-mode), exactly as for snapshot anomalies.
  • Early tolerance (emergent) — start-states are normal (dense) via the maturity encoding, so CVaDE does not false-alarm at trajectory starts.

The framework (whitened residual, nearest-mode NLL, C1 generation, C2 basin, C3 ensemble, C4 risk control) runs unchanged on the history-aware $\xi_t$.

6. Training pipeline

  1. Train LDT on normal (multi-resolution masked reconstruction of the past) → pass the memory-horizon probe → freeze.
  2. Compute $c_t$ for all data; form $\xi_t=[\phi(W_t)\oplus c_t]$.
  3. Train CVaDE on normal $\xi_t$ (VaDE + anti-collapse); fit whitener, C3 ensemble, C4 thresholds on normal.
  4. Detect with CVaDE. (No separate LDT score.)

7. Grounding

Composes established ideas: masked autoencoders / denoising sequence models; multi-timescale architectures (clockwork RNN, dilated-TCN/WaveNet, S4/Mamba); slow-feature analysis; representation-learning-as-feature -extractor (pretrain → freeze). Novel angle: a purely history-encoding twin feeding a MIIM mode detector whose components (C1–C4) do all anomaly reasoning on a history-aware input.

Open questions / to discuss

  • Backbone: dilated-TCN vs SSM (Mamba/S4) vs clockwork RNN — memory vs simplicity vs dependencies.
  • Masking scheme: random-step masking vs block masking vs "reconstruct downsampled" per scale.
  • Should the short branch also just pass through recent raw features (is it redundant with the window)?
  • Dimensionality budget of $c^{\text{short/med/long}}$ vs the window, so none swamps in $\xi_t$.
  • Two-stage frozen vs later end-to-end fine-tuning of LDT+CVaDE.
  • Non-stationarity across runs / sensor drift — per-run normalisation vs a learned "session" regime.
