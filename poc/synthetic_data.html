Synthetic MIIM data: model & generation process

One controlled generator with ground truth: a unified MIIM benchmark that manufactures normal operation carrying every assumed phenomenon (A1–A10), plus a labelled fault taxonomy, so each detection mechanism can be validated where real data cannot.

Why synthetic. Genuine CPS faults are rare, unlabelled, and mixed with distribution shift; real testbeds (HAI/SKAB) turned out to contain only snapshot-detectable anomalies. A controlled generator gives (i) ground-truth modes and anomaly types, (ii) a non-circular oracle for labels, and (iii) the ability to inject history-dependent anomalies that no real benchmark we found exposes.

0. Data assumptions (A1–A10)

Everything downstream rests on a single generative picture of normal CPS operation. Let $x\in\mathbb R^{d}$ be one channel-vector and $p(x)$ the normal law. Each mode $k$ carries an intrinsic regime coordinate $u\in\mathbb R^{r}$ (per-mode spread $\sigma_k$) mapped to the $d$ channels by a smooth mode map $g_k$ (specified in §2). The table gives each assumption's rationale and scope (global, or mode-dependent so it holds on some modes and not others); its precise statement and how the generator realises it are in the recipe (§2). The generator satisfies exactly these, and the detector is designed against them.

IDassumptionwhy we assume it (CPS origin)scope

A2Mode Explosion (exponential mode growth) a modern CPS is built from many interacting subsystems, and each added element (or new context) multiplies the reachable regimes, so the number of modes grows exponentially with system complexity. global
A3Hard Envelopes (bounded modes) controllers mostly hold each regime within setpoint bands and actuator limits; the space between regimes is usually only passed through, rarely dwelt in. A4Thin Fringes (intra-mode sparsity) within a single mode the system's presence is non-uniform: it dwells near a typical operating point and reaches the edges of the regime only occasionally, so each normal mode holds legitimately low-probability regions, most often near its boundary. global
A5Few Levers (low intrinsic dimension) the independent control variables are far fewer than the observed variables (the channels, or a whole window); the rest follow from them, deterministically or probabilistically. global

Two standing qualifiers apply throughout: the sample is massive ($N$ large) and the mixture is stationary (the $\pi_k$ and $g_k$ do not drift within a run). A1–A8 shape each window (instantaneous structure); A9–A10 shape the trajectory (dynamics and history); one unified generator (§1) produces a single time-series carrying all ten. Physical joint coupling (marginally in-range on every channel yet jointly off-manifold) is not a separate row: it is inherent in the curved, cross-channel $g_k$ of §2.

Mode-dependent by design (a built-in ablation). A8, A9, A10 and per-mode curvature ($\kappa_k$, Step 2) hold on some modes and not others: some modes drive the full heterogeneous channel mix, others a homogeneous subset; some are curved ($\kappa_k>0$), others flat ($\kappa_k=0$); some evolve on several time scales, others one; some are valid only given the right trajectory, others are always valid. The same split applies to anomalies (some sit in the simple subset, some in the complex). Because a phenomenon and its absence both live in one dataset, we can read off, per phenomenon, exactly which normal modes and which anomalies a given detector models or misses.

1. One unified generator

A single streaming generator produces a massive multivariate time-series of normal CPS operation carrying every phenomenon A1–A10 (the MIIM property: Massive, Implicit, Imbalanced, Multimodal, plus CPS structure), together with a labelled test stream spanning the full anomaly taxonomy (Step 7). The recipe is seven steps: Steps 1–6 build the normal series, Step 7 injects the labelled faults; each is tagged with the assumptions it realises, so the data model and the assumption list stay one-to-one.

Shape & notation. A single timestep is a channel-vector $x_t\in\mathbb R^{d}$ ($d$ = the number of channels, of mixed type per Step 3; default $d=24$). The generator emits a length-$T$ multivariate time series $X=(x_1,\dots,x_T)\in\mathbb R^{T\times d}$ with $T$ large (a normal train series and a labelled test series). Detection runs per window: overlapping windows $x_{t-W+1:t}\in\mathbb R^{W\times d}$ of length $W$ and stride $S$ (default $W=40$, $S=20$), each carrying a window label $y_t\in\{0,1\}$ plus its dominant mode and anomaly type. So the shapes are: series $T\times d$, window $W\times d$.

2. The recipe, step by step

Steps 1–6 produce the normal time-series; Step 7 injects the labelled test faults. Each step is tagged with the assumptions it realises.

Step 1: Compositional mode space A1A2A6

In plain terms: picture the machine as a handful of dials, each with a few settings; a mode is one full combination of settings. A few dials already make a huge number of combinations, and the machine spends most of its time in only a few of them.

The system is $n_{\text{sub}}$ interacting subsystems, each with a few discrete states. A global mode is a tuple of sub-states $(s_1,\dots,s_{n_{\text{sub}}})$, so the number of candidate modes $K=\prod_j|S_j|$ grows exponentially with the number of subsystems (A2). Not every combination is physically reachable, because subsystems constrain one another (an "engine-off" state cannot co-occur with "high RPM"; a closed valve cannot co-occur with "high downstream flow"). A compatibility relation (a set of forbidden sub-state co-occurrences, sampled once and then fixed) prunes the contradictory tuples; the survivors are the mode vocabulary, and the only modes the transition graph (Step 4) can ever visit. The result is far smaller than $\prod_j|S_j|$ yet still exponentially large: massive but not arbitrary. The reachable modes carry Zipf stationary frequencies $\pi_k\propto\mathrm{rank}(k)^{-\beta}$, shuffled so identity is decoupled from frequency and the rare tail carries $O(1)$ mass (A6); the normal law is then the mixture $p(x)=\sum_k\pi_k\,p_k(x)$ over the reachable modes (A1).

Exact forms & example. frequencies $\pi_k=\dfrac{\mathrm{rank}(k)^{-\beta}}{\sum_j \mathrm{rank}(j)^{-\beta}}$, then the rank$\to$mode assignment is shuffled. E.g. $n_{\text{sub}}=4$ subsystems with $|S_j|\in\{3,4,5\}$ give up to $\prod_j|S_j|=180$ candidate modes; a compatibility relation forbidding, say, half the pairwise sub-state co-occurrences leaves on the order of $70$ reachable modes. With $\beta=2$ the most common mode is $\approx 2^2=4\times$ the second and the tail decays as $\mathrm{rank}^{-2}$, so the rarest modes appear only a handful of times in a length-$T$ run.

Step 2: Per-mode geometry A3A4A5

In plain terms: each mode is an oriented blob in latent space (the figure at the end of §0): a centre, a few directions it can stretch along, and a hard rim. Most blobs are simple (flat); a few are bent, which is the hard case for a detector.

Base: a clipped low-rank Gaussian. Each mode $k$ has a low-dimensional regime coordinate $u\in\mathbb R^{r}$ ($r\ll d$; e.g. $r=3$, $d=24$) and a signature

$$g_k(u)=\mu_k+B_k u,\qquad u\sim\mathcal N(0,\sigma_k^2 I_r)\ \text{clipped to}\ \pm2\sigma_k,\qquad x=g_k(u)+\varepsilon,$$

where $\mu_k\in\mathbb R^{d}$ is the mode centre and $B_k\in\mathbb R^{d\times r}$ a low-rank loading whose columns are the mode's few "levers" (its principal stretch directions). This is a mixture of factor analyzers: mode $k$ is Gaussian with covariance $\sigma_k^2 B_kB_k^\top+\Sigma_\varepsilon$ (Step 3's noise; $\sigma_k$ sets the mode's overall scale, $B_k$'s columns its directions). The clip gives a bounded patch with a hard rim (A3); the Gaussian on $u$ concentrates mass at the centre and thins toward the rim (A4); and $r\ll d$ makes it a thin, oriented ellipsoid (A5, few levers): exactly the blobs in the §0 figure.

Curvature knob $\kappa_k$ (mode-dependent). A flat subspace is too easy: per-mode PCA reconstructs it exactly, so a linear detector already wins, and the region "between two modes" can look like a valid third mode. So we add an optional bend,

$$g_k(u)=\mu_k+B_k u+\kappa_k\,\tanh\!\big(1.3\,(B_k u)^{\leftarrow}\big),$$

with a per-mode strength $\kappa_k$: $\kappa_k=0$ recovers the plain clipped Gaussian (linear, PCA-solvable, a control); $\kappa_k>0$ (e.g. $0.35$) bends it into a genuinely nonlinear manifold whose between-modes region falls off every mode, so the over-interpolation false-negative (E2) appears and a nonlinear encoder is needed. The channel reversal $(\cdot)^{\leftarrow}$ makes the bend non-local (channel $c$ couples to channel $d{-}c$), so the joint constraint hides from any single channel or adjacent pair. Making $\kappa_k$ mode-dependent adds a curvature axis to the built-in ablation (alongside A8/A9/A10): some modes flat, some curved, so one dataset shows whether the nonlinear encoder earns its keep only where it should. Figure 2 renders the knob as a 2D density heatmap of a single mode: at $\kappa_k=0$ the mode is a straight, oriented ellipse (a flat subspace); as $\kappa_k$ grows its high-density ridge bows into a banana, so the peak follows a curve and the between-modes region drops off every mode.

Example. $\kappa_k=0$ gives the dashed line (a flat low-rank subspace, PCA fits it exactly); $\kappa_k=0.35$ gives the green arc. Normal windows lie on it (dense at the centre, sparse at the $\pm2\sigma$ ends); the red point is in every channel's range yet off the arc.

Step 3: Typed, heterogeneous channels A8

In plain terms: the channels are a mixed bag: fine sensors, on/off actuators, integer state flags, held-then-stepped setpoints, and one slow fuel-gauge. Some modes use the whole bag, others just a plain subset.

The $d$ channels are a mix of variable types: each renders its underlying $g_k(u)$ value differently, with its own scale, resolution $\delta_c$, and correlated heteroscedastic noise $\Sigma_\varepsilon=W_\varepsilon W_\varepsilon^\top+\operatorname{diag}(s^2)$ ($W_\varepsilon$ a low-rank loading matrix, distinct from the window length $W$):

  • sensor: the continuous value, finely quantized;
  • control / actuator: held piecewise-constant within a mode, coarse or near-binary;
  • discrete state: thresholded to an integer subsystem-state indicator;
  • setpoint: held, then stepped only at a transition;
  • accumulator: the exception, a slow integrator that overrides its channel (Step 5), not a rendering of $g_k$.

The noise is deliberately correlated (via the low-rank $W_\varepsilon$), not white: a decoder cannot reconstruct noise, so residuals inherit $\Sigma_\varepsilon$, which is the structure a whitened (Mahalanobis / Ledoit-Wolf) residual score exploits and a plain sum-of-squares cannot. Heteroscedasticity ($\operatorname{diag}(s^2)$) and per-channel quantization $\delta_c$ make the channels genuinely unequal, so which channel a fault touches matters.

Dimensions (all matrices, across the recipe). Per timestep $x,\ \mu_k,\ g_k(u),\ \varepsilon,\ s\in\mathbb R^{d}$ ($d$ channels); regime coordinate $u\in\mathbb R^{r}$ and loading $B_k\in\mathbb R^{d\times r}$ (Step 2, $r\ll d$); noise loading $W_\varepsilon\in\mathbb R^{d\times q}$ of rank $q\ll d$, so $\Sigma_\varepsilon\in\mathbb R^{d\times d}$; the per-channel scalars $\delta_c,\ \theta_c,\ \sigma_c\in\mathbb R$ for $c=1,\dots,d$. At the mode level the prior $\pi\in\Delta^{K}$ and the transition matrix $P\in\mathbb R^{K\times K}$ (Steps 1, 4); a window is $x_{t-W+1:t}\in\mathbb R^{W\times d}$.

Exact renderings of a channel's raw value $v_c=g_k(u)_c+\varepsilon_c$, $\varepsilon\sim\mathcal N(0,\Sigma_\varepsilon)$: sensor $\hat v_c=\delta_c\,\mathrm{round}(v_c/\delta_c)$ with small $\delta_c$; actuator the same with large $\delta_c$ (coarse); discrete state $\mathbb 1[v_c>\theta_c]$ (or a few integer levels); setpoint holds the value taken at mode entry until the next transition. E.g. a fine sensor uses $\delta_c=0.02\,\sigma_c$, a near-binary switch $\delta_c=0.8\,\sigma_c$.

Which types are active is mode-dependent: some modes drive the full mix, others only a homogeneous subset (a built-in control for A8).

Step 4: Trajectory dynamics A9

In plain terms: over time the machine walks from mode to mode, but only along allowed routes, pausing a while in each. Channels react at different speeds (temperature lags, RPM snaps); some modes juggle several speeds, some just one.

Normal operation is a walk over the reachable modes: a sparse transition graph (a few allowed successors per mode, Dirichlet-weighted; every other edge forbidden, which is what makes a forbidden jump an anomaly in Step 5) with per-mode dwell times log-uniform in $[15,220]$ (log-uniform so durations spread evenly across an order of magnitude: some regimes brief, some long, a temporal-scale mix). Within a dwell the regime coordinate follows an Ornstein-Uhlenbeck walk (mean-reverting to the mode centre, clipped to stay in bounds; the coefficient $0.85$ sets the within-mode memory, i.e. the autocorrelation length),

$$u_t=0.85\,u_{t-1}+0.15\,\varepsilon_t,\quad\varepsilon_t\sim\mathcal N(0,\sigma_k^2 I_r),\qquad u_t\leftarrow\operatorname{clip}(u_t,\pm2\sigma_k),$$

emitted through $g_k$; each channel is then smoothed with its own time constant $x^{(c)}_t=\alpha_c x^{(c)}_t+(1-\alpha_c)x^{(c)}_{t-1}$, $\alpha_c\in[0.05,0.95]$ (slow / inertial vs fast: the multiscale clocks). Whether a mode is multiscale is mode-dependent: some modes span several time constants, others a single one.

Exact forms. transition row over the allowed successors $P_{k,\cdot}\sim\mathrm{Dirichlet}(\mathbf1)$ (zero on forbidden edges); dwell $D\sim\big\lfloor e^{\,\mathcal U(\ln 15,\ln 220)}\big\rfloor$; per-channel constant $\alpha_c\sim\mathcal U(0.05,0.95)$. A slow channel ($\alpha_c=0.05$) lags a step change by $\sim\!1/\alpha_c\approx20$ samples; a fast one ($\alpha_c=0.9$) tracks it within $\sim\!1$.

Step 5: History-conditioned normality A10

In plain terms: whether a reading is fine can depend on the story so far. A low fuel-gauge is normal after a long haul but alarming right after refuelling; a settled state is fine if reached by a legal route, suspicious if you jumped there illegally. Only some modes work this way.

Two mechanisms make the same instantaneous state normal or anomalous depending on the past:

  • Accumulator. An integrator channel tracks, e.g., state-of-charge, $a_t=\operatorname{clip}(a_{t-1}-\ell(\text{mode}_t)+\rho_t,\,0,\,1)$ with $\ell$ a mode-dependent load and $\rho_t$ a recharge; a low $a_t$ is normal iff preceded by high cumulative load.
  • Transition graph. A settled state is normal iff reached by an allowed path; the identical values reached via a forbidden predecessor are a fault.

Path-dependence is mode-dependent: only some modes are history-conditioned; the rest are always-valid controls, so the dataset holds matched with / without pairs.

Example. load $\ell=0.02$ in cruise, $0.08$ under heavy load, recharge $\rho=0.05$ only in a "charging" mode. A settled window at $a_t=0.1$ is normal after a long heavy-load dwell but a fault after a fresh start; independently, a valid mode reached from a forbidden predecessor is a fault by the graph rule. These are the two Step-7 history anomalies.

Step 6: Implicit, and framing A7

In plain terms: the trainer only ever sees the raw numbers, never which mode or setting produced them, and it sees a great many of them.

No mode or subsystem-state label is emitted for training (A7): only the raw channel stream, labels kept for evaluation. The stream is standardised on train statistics only (never the test set, so there is no leakage and the detector sees exactly the normalisation it would at deployment) and windowed (window $W$, stride $S$; per-window label by majority, plus the dominant anomaly type and mode). The sample is massive, which A1's mixture and A2's mode count require to populate even the rare tail.

Exact forms. standardise $z_t=(x_t-\mu_{\text{tr}})/\sigma_{\text{tr}}$ ($\mu_{\text{tr}},\sigma_{\text{tr}}$ the per-channel train mean and std); take windows $z_{t-W+1:t}$ every $S$ steps; window label $y=\mathbb 1[\text{fraction of anomalous steps}>\tfrac12]$, with its majority mode and dominant anomaly type.

Step 7: Anomaly generation (test stream only) A3A4A10

In plain terms: the training data is all-normal; only the test stream gets faults, six kinds, ranging from obvious (way out of range) to subtle (a perfectly ordinary reading reached by an impossible history).

Faults are injected only into the test series. $c_k$ is mode $k$'s centroid in signal space, $D$ the dwell length. Each family targets a specific detection error (E1 rare-mode false-positive, E2 over-interpolation false-negative, E3 label-free calibration):

  • snapshot OOD: a point far outside every mode, $x=c_{\text{near}}+1.8\,(x_0-c_{\text{near}})$, $x_0\sim\mathcal U(\text{wide box})$, $c_{\text{near}}$ its nearest centroid (pushed further out to guarantee it is off).
  • snapshot pocket: a short between-two-modes segment $x=(1-t)\,c_a+t\,c_b$, $t\sim\mathcal U(0.35,0.65)$: the over-interpolation false-negative (E2).
  • snapshot drift: inside a dwell add an un-clipped ramp $u_t\leftarrow u_t+g\,(k/D)$ so the coordinate leaves $\pm2\sigma_k$ and the values leave the manifold.
  • contextual wrong-for-regime: emit another mode's signature $g_a(u)$ while the trajectory is in mode $b$: values valid for $a$, invalid given context $b$.
  • history bad_transition: jump a forbidden edge $a\!\to\!b$ and label the windows settled inside the valid target $b$ (after the straddle), snapshot-normal, so only the trajectory channel can catch them.
  • history premature_depletion: drive the accumulator low ($a_t\!\downarrow$) without the preceding heavy-load dwell: snapshot-normal, only the integral of the past is inconsistent.
Why the history families matter. On the current generator a snapshot detector (VaDE, per-type TPR @5% FPR) reaches pocket 0.72, drift 0.91, but settled bad_transition 0.057 $\approx$ the false-positive floor: it is blind to history-dependent faults. bad_transition and premature_depletion are the two controlled cases that isolate the trajectory model (B), which no real testbed we found provides.

Putting it together: the coverage walk

In plain terms: build the machine and its rule-book once, then take a long walk through it, deliberately detouring so every rare corner and every kind of fault gets enough visits to be measured.

Two phases. (A) Instantiate the system once: sample the valid modes (Step 1), the dynamics graph (valid edges including a few low-probability ones, plus the forbidden edges; Step 4), and the per-mode geometry, channel types and sensor parameters (Steps 2, 3, 5). (B) Generate streams by walking the graph, rendering each step (Steps 2–3), evolving the dynamics (Step 4), and injecting faults into the test stream (Step 7).

A natural walk follows the Zipf frequencies and the graph, which is realistic but starves the tail: rare modes, rare-but-valid edges and the mode fringes are exactly what we must measure yet are barely visited. So the walk is coverage-driven: keep the natural walk for realism, but overlay forced excursions with per-variant quotas so each case reaches a measurable count, labelled by construction (we know when we took a forbidden edge, pushed to a fringe, or injected a fault):

Quotas make the rare-valid cases (modes, edges, fringe) numerous enough to measure the rare-mode false-alarm rate (E1), while every anomaly family has a controlled, labelled presence.

3. Non-circular labelling & difficulty

Labels never use the detector. A true-normal oracle holds a large fresh normal sample from the same modes/frame; a candidate is a genuine anomaly iff its $\kappa$-NN distance to that reference exceeds the $99^{\text{th}}$ percentile of normal self-distance (self-match dropped). A model-agnostic distance to the nearest mode uses a pooled within-mode Mahalanobis $\sigma(x)=\min_k\sqrt{(x-\bar x_k)^\top S^{-1}(x-\bar x_k)}$; the normal envelope $E=Q_{0.99}(\sigma)$ sets difficulty bands, so a fault at $\sigma(x)/E\in[1,2]$ is "just outside" (hard).

5. What it enables & build status

  • ground-truth modes, subsystem states, and anomaly types for per-type diagnostics;
  • a non-circular oracle and model-agnostic distance / envelope for detector-independent labels and difficulty bands;
  • two history-dependent families (bad_transition, premature_depletion) that isolate the trajectory model;
  • label-free hooks: C1-style generated anomalies for model selection, and the memory-horizon probe for B.
Build status (honest). Today's code realises A1, A3–A7, A9 and the discrete A10 (data.py, temporal_data.py). This unified recipe adds the three not yet in code: compositional / massive modes (A2), typed channels (A8: control, state, setpoint), and the accumulator history family (A10, continuous). It is a single new generator; the snapshot-blindness check above is re-run to confirm the baselines hold.
