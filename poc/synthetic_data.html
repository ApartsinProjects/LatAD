Synthetic MIIM data: model & generation process

One controlled generator with ground truth: a unified MIIM benchmark that manufactures normal operation carrying every assumed phenomenon (A1–A10), plus a labelled fault taxonomy, so each detection mechanism can be validated where real data cannot.

Why synthetic. Genuine CPS faults are rare, unlabelled, and mixed with distribution shift; real testbeds (HAI/SKAB) turned out to contain only snapshot-detectable anomalies. A controlled generator gives (i) ground-truth modes and anomaly types, (ii) a non-circular oracle for labels, and (iii) the ability to inject history-dependent anomalies that no real benchmark we found exposes.

0. Data assumptions (A1–A10)

Everything downstream rests on a single generative picture of normal CPS operation. Let $x\in\mathbb R^{d}$ be one channel-vector and $p(x)$ the normal law. Each mode $k$ carries an intrinsic regime coordinate $u\in\mathbb R^{r}$ (per-mode spread $\sigma_k$) mapped to the $d$ channels by a smooth mode map $g_k$ (specified in §2). The table gives each assumption's rationale and scope (global, or mode-dependent so it holds on some modes and not others); its precise statement and how the generator realises it are in the recipe (§2). The generator satisfies exactly these, and the detector is designed against them.

IDassumptionwhy we assume it (CPS origin)scope
A1Regime Mixture (multimodality) a plant or vehicle cycles through discrete operating regimes (idle, cruise, load steps, maneuvers); normal is their union, not one blob. global
A2Mode Explosion (exponential mode growth) a modern CPS is built from many interacting subsystems, and each added element (or new context) multiplies the reachable regimes, so the number of modes grows exponentially with system complexity. global
A3Hard Envelopes (bounded modes) controllers mostly hold each regime within setpoint bands and actuator limits; the space between regimes is usually only passed through, rarely dwelt in. global
A4Thin Fringes (intra-mode sparsity) within a single mode the system's presence is non-uniform: it dwells near a typical operating point and reaches the edges of the regime only occasionally, so each normal mode holds legitimately low-probability regions, most often near its boundary. global
A5Few Levers (low intrinsic dimension) the independent control variables are far fewer than the observed variables (the channels, or a whole window); the rest follow from them, deterministically or probabilistically. global
A6Heavy Tail (Zipf imbalance) machines spend most of the record in a few steady regimes; startup, shutdown and rare maneuvers occupy very little of it; so an infrequent but perfectly normal point is easily mistaken for an anomaly. global
A7Hidden Regimes (latent modes) the true regimes are an emergent artifact of complex system design and coupling, so the actual modes can rarely be recovered exactly; the mode label is not logged either, so modes must be discovered (approximately) from raw streams. global
A8Mixed Signals (heterogeneous variables) the CPS bus carries a mix of variable types (sensor readings, actuator / control signals, discrete system states, setpoints), each with its own range, resolution, and correlated noise; not sensors alone. mode-dependent
A9Many Clocks (multiscale dynamics) physical inertia sets per-channel time constants (thermal slow, electrical fast); operating procedure makes only some regime→regime transitions legal, each with a characteristic dwell. mode-dependent · temporal
A10Path Dependence (history-conditioned normality) a fault is defined relative to the operational history: the same instantaneous state can be normal or faulty depending on how the system got there (a low battery is fine after intensive operation, but anomalous right after a fresh start / full charge). mode-dependent · temporal

Two standing qualifiers apply throughout: the sample is massive ($N$ large) and the mixture is stationary (the $\pi_k$ and $g_k$ do not drift within a run). A1–A8 shape each window (instantaneous structure); A9–A10 shape the trajectory (dynamics and history); one unified generator (§1) produces a single time-series carrying all ten. Physical joint coupling (marginally in-range on every channel yet jointly off-manifold) is not a separate row: it is inherent in the curved, cross-channel $g_k$ of §2.

Mode-dependent by design (a built-in ablation). A8, A9 and A10 hold on some modes and not others: some modes drive the full heterogeneous channel mix, others a homogeneous subset; some evolve on several time scales, others one; some are valid only given the right trajectory, others are always valid. The same split applies to anomalies (some sit in the simple subset, some in the complex). Because a phenomenon and its absence both live in one dataset, we can read off, per phenomenon, exactly which normal modes and which anomalies a given detector models or misses.

1. One unified generator

A single streaming generator produces a massive multivariate time-series of normal CPS operation carrying every phenomenon A1–A10 (the MIIM property: Massive, Implicit, Imbalanced, Multimodal, plus CPS structure), together with a labelled test stream spanning the full anomaly taxonomy (§3). The recipe is six steps; each is tagged with the assumptions it realises, so the data model and the assumption list stay one-to-one.

Shape & notation. A single timestep is a channel-vector $x_t\in\mathbb R^{d}$ ($d$ = the number of channels, of mixed type per Step 3; default $d=24$). The generator emits a length-$T$ multivariate time series $X=(x_1,\dots,x_T)\in\mathbb R^{T\times d}$ with $T$ large (a normal train series and a labelled test series). Detection runs per window: overlapping windows $x_{t-W+1:t}\in\mathbb R^{W\times d}$ of length $W$ and stride $S$ (default $W=40$, $S=20$), each carrying a window label $y_t\in\{0,1\}$ plus its dominant mode and anomaly type. So the shapes are: series $T\times d$, window $W\times d$.

2. The recipe, step by step

Six steps produce one normal time-series; each step is tagged with the assumptions it realises.

Step 1: Compositional mode space A1A2A6

In plain terms: picture the machine as a handful of dials, each with a few settings; a mode is one full combination of settings. A few dials already make a huge number of combinations, and the machine spends most of its time in only a few of them.

The system is $n_{\text{sub}}$ interacting subsystems, each with a few discrete states. A global mode is a tuple of sub-states $(s_1,\dots,s_{n_{\text{sub}}})$, so the number of candidate modes $K=\prod_j|S_j|$ grows exponentially with the number of subsystems (A2). Not every combination is physically reachable, because subsystems constrain one another (an "engine-off" state cannot co-occur with "high RPM"; a closed valve cannot co-occur with "high downstream flow"). A compatibility relation (a set of forbidden sub-state co-occurrences, sampled once and then fixed) prunes the contradictory tuples; the survivors are the mode vocabulary, and the only modes the transition graph (Step 4) can ever visit. The result is far smaller than $\prod_j|S_j|$ yet still exponentially large: massive but not arbitrary. The reachable modes carry Zipf stationary frequencies $\pi_k\propto\mathrm{rank}(k)^{-\beta}$, shuffled so identity is decoupled from frequency and the rare tail carries $O(1)$ mass (A6); the normal law is then the mixture $p(x)=\sum_k\pi_k\,p_k(x)$ over the reachable modes (A1).

Step 2: Per-mode geometry A3A4A5

In plain terms: inside a mode only a few hidden knobs move, and every channel is a smooth function of them. The machine mostly sits near the middle of a mode and only rarely drifts to its edge, never leaving the allowed patch.

Each mode $k$ has a low-dimensional regime coordinate $u\in\mathbb R^{r}$ ($r\ll d$; e.g. $r=3$, $d=24$), mapped to the channels by a smooth, curved, mode-specific manifold

$$g_k(u)=A_k u+b_k+0.35\,\tanh\!\big(1.3\,(A_k u+b_k)^{\leftarrow}\big),$$

with $A_k\in\mathbb R^{d\times r}$, $b_k\in\mathbb R^{d}$ fixed per mode. The linear part $A_k u+b_k$ places and orients the mode; the $\tanh$ term bends it into a curved manifold (strength $0.35$, so the curvature is mild but real); and $(\cdot)^{\leftarrow}$, a channel reversal, makes the coupling non-local (channel $c$ interacts with channel $d{-}c$), so the joint constraint shows in no single channel or adjacent pair: a point can be marginally in-range on every channel yet jointly off-manifold. $g_k$ is additive over subsystems (each contributes a channel block), so composition shows in signal space. The regime coordinate is concentrated and clipped: $u\sim\mathcal N(0,\sigma_k^2 I_r)$ truncated to $\pm2\sigma_k$, giving bounded modes with hard edges (A3) whose density concentrates at the centre and thins toward the boundary (A4).

Step 3: Typed, heterogeneous channels A8

In plain terms: the channels are a mixed bag: fine sensors, on/off actuators, integer state flags, held-then-stepped setpoints, and one slow fuel-gauge. Some modes use the whole bag, others just a plain subset.

The $d$ channels are a mix of variable types: each renders its underlying $g_k(u)$ value differently, with its own scale, resolution $\delta_c$, and correlated heteroscedastic noise $\Sigma_\varepsilon=W_\varepsilon W_\varepsilon^\top+\operatorname{diag}(s^2)$ ($W_\varepsilon$ a low-rank loading matrix, distinct from the window length $W$):

  • sensor: the continuous value, finely quantized;
  • control / actuator: held piecewise-constant within a mode, coarse or near-binary;
  • discrete state: thresholded to an integer subsystem-state indicator;
  • setpoint: held, then stepped only at a transition;
  • accumulator: the exception, a slow integrator that overrides its channel (Step 5), not a rendering of $g_k$.

The noise is deliberately correlated (via the low-rank $W_\varepsilon$), not white: a decoder cannot reconstruct noise, so residuals inherit $\Sigma_\varepsilon$, which is the structure a whitened (Mahalanobis / Ledoit-Wolf) residual score exploits and a plain sum-of-squares cannot. Heteroscedasticity ($\operatorname{diag}(s^2)$) and per-channel quantization $\delta_c$ make the channels genuinely unequal, so which channel a fault touches matters.

Which types are active is mode-dependent: some modes drive the full mix, others only a homogeneous subset (a built-in control for A8).

Step 4: Trajectory dynamics A9

In plain terms: over time the machine walks from mode to mode, but only along allowed routes, pausing a while in each. Channels react at different speeds (temperature lags, RPM snaps); some modes juggle several speeds, some just one.

Normal operation is a walk over the reachable modes: a sparse transition graph (a few allowed successors per mode, Dirichlet-weighted; every other edge forbidden, which is what makes a forbidden jump an anomaly in Step 5) with per-mode dwell times log-uniform in $[15,220]$ (log-uniform so durations spread evenly across an order of magnitude: some regimes brief, some long, a temporal-scale mix). Within a dwell the regime coordinate follows an Ornstein-Uhlenbeck walk (mean-reverting to the mode centre, clipped to stay in bounds; the coefficient $0.85$ sets the within-mode memory, i.e. the autocorrelation length),

$$u_t=0.85\,u_{t-1}+0.15\,\varepsilon_t,\quad\varepsilon_t\sim\mathcal N(0,\sigma_k^2 I_r),\qquad u_t\leftarrow\operatorname{clip}(u_t,\pm2\sigma_k),$$

emitted through $g_k$; each channel is then smoothed with its own time constant $x^{(c)}_t=\alpha_c x^{(c)}_t+(1-\alpha_c)x^{(c)}_{t-1}$, $\alpha_c\in[0.05,0.95]$ (slow / inertial vs fast: the multiscale clocks). Whether a mode is multiscale is mode-dependent: some modes span several time constants, others a single one.

Step 5: History-conditioned normality A10

In plain terms: whether a reading is fine can depend on the story so far. A low fuel-gauge is normal after a long haul but alarming right after refuelling; a settled state is fine if reached by a legal route, suspicious if you jumped there illegally. Only some modes work this way.

Two mechanisms make the same instantaneous state normal or anomalous depending on the past:

  • Accumulator. An integrator channel tracks, e.g., state-of-charge, $a_t=\operatorname{clip}(a_{t-1}-\ell(\text{mode}_t)+\rho_t,\,0,\,1)$ with $\ell$ a mode-dependent load and $\rho_t$ a recharge; a low $a_t$ is normal iff preceded by high cumulative load.
  • Transition graph. A settled state is normal iff reached by an allowed path; the identical values reached via a forbidden predecessor are a fault.

Path-dependence is mode-dependent: only some modes are history-conditioned; the rest are always-valid controls, so the dataset holds matched with / without pairs.

Step 6: Implicit, and framing A7

In plain terms: the trainer only ever sees the raw numbers, never which mode or setting produced them, and it sees a great many of them.

No mode or subsystem-state label is emitted for training (A7): only the raw channel stream, labels kept for evaluation. The stream is standardised on train statistics only (never the test set, so there is no leakage and the detector sees exactly the normalisation it would at deployment) and windowed (window $W$, stride $S$; per-window label by majority, plus the dominant anomaly type and mode). The sample is massive, which A1's mixture and A2's mode count require to populate even the rare tail.

3. Anomaly taxonomy (test stream)

Faults span snapshot to history-dependent, so every detection error has a controlled case:

  • snapshot OOD: far outside every mode.
  • snapshot pocket: a short between-two-modes segment $x=(1-t)c_a+t\,c_b$, $t\in[0.35,0.65]$ (the over-interpolation false-negative).
  • snapshot drift: the OU coordinate pushed out of bounds (bypasses the clip), so values leave the manifold.
  • contextual wrong-for-regime: a value valid in some mode, invalid for the current one.
  • history bad_transition: a forbidden mode→mode jump; the labelled windows are settled inside the valid target mode, snapshot-normal, so only the trajectory channel can catch them.
  • history premature_depletion: the accumulator is low without the load history that would justify it: snapshot-normal, only the integral of the past is inconsistent.
Why the history families matter. On the current generator a snapshot detector (VaDE, per-type TPR @5% FPR) reaches pocket 0.72, drift 0.91, but settled bad_transition 0.057 $\approx$ the false-positive floor: it is blind to history-dependent faults. bad_transition and premature_depletion are the two controlled cases that isolate the trajectory model (B), which no real testbed we found provides.

4. Non-circular labelling & difficulty

Labels never use the detector. A true-normal oracle holds a large fresh normal sample from the same modes/frame; a candidate is a genuine anomaly iff its $\kappa$-NN distance to that reference exceeds the $99^{\text{th}}$ percentile of normal self-distance (self-match dropped). A model-agnostic distance to the nearest mode uses a pooled within-mode Mahalanobis $\sigma(x)=\min_k\sqrt{(x-\bar x_k)^\top S^{-1}(x-\bar x_k)}$; the normal envelope $E=Q_{0.99}(\sigma)$ sets difficulty bands, so a fault at $\sigma(x)/E\in[1,2]$ is "just outside" (hard).

6. What it enables & build status

  • ground-truth modes, subsystem states, and anomaly types for per-type diagnostics;
  • a non-circular oracle and model-agnostic distance / envelope for detector-independent labels and difficulty bands;
  • two history-dependent families (bad_transition, premature_depletion) that isolate the trajectory model;
  • label-free hooks: C1-style generated anomalies for model selection, and the memory-horizon probe for B.
Build status (honest). Today's code realises A1, A3–A7, A9 and the discrete A10 (data.py, temporal_data.py). This unified recipe adds the three not yet in code: compositional / massive modes (A2), typed channels (A8: control, state, setpoint), and the accumulator history family (A10, continuous).
