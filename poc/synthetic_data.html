Synthetic MIIM data: model & generation process

Two controlled generators with ground truth: a static MIIM benchmark (data.py) and a temporal MIIM benchmark (temporal_data.py). Both manufacture normal operation with the phenomena the framework assumes, plus labelled faults, so each detection mechanism can be validated where real data cannot.

Why synthetic. Genuine CPS faults are rare, unlabelled, and mixed with distribution shift; real testbeds (HAI/SKAB) turned out to contain only snapshot-detectable anomalies. A controlled generator gives (i) ground-truth modes and anomaly types, (ii) a non-circular oracle for labels, and (iii) the ability to inject history-dependent anomalies that no real benchmark we found exposes.

0. Data assumptions (A1–A10)

Everything downstream rests on a single generative picture of normal CPS operation. Let $x\in\mathbb R^{d}$ be one channel-vector and $p(x)$ the normal law. Each mode $k$ carries an intrinsic regime coordinate $u\in\mathbb R^{r}$ (per-mode spread $\sigma_k$) mapped to the $d$ channels by a smooth mode map $g_k$ (fully specified in §2). The assumptions, stated precisely and compactly; the generators are built to satisfy exactly these, and the detector is designed against them.

Two standing qualifiers apply throughout: the sample is massive ($N$ large) and the mixture is stationary (the $\pi_k$ and $g_k$ do not drift within a run). A1–A8 are static (both generators); A9–A10 (dynamics and history) are the temporal generator only. Physical joint coupling (marginally in-range on every channel yet jointly off-manifold) is not a separate row — it is inherent in the curved, cross-channel $g_k$ of §2.

2. Common core — a mode as a bounded nonlinear manifold

Both generators share the mode model. There are $K$ operating modes; mode $k$ has an intrinsic regime coordinate $u\in\mathbb R^{r}$ ($r=3$) mapped to $d=24$ channels by a smooth, mode-specific nonlinear embedding

$$g_k(u)=A_k u+b_k+0.35\,\tanh\!\big(1.3\,(A_k u+b_k)^{\leftarrow}\big),$$

where $A_k\in\mathbb R^{d\times r}$, $b_k\in\mathbb R^{d}$ are fixed per mode (a deterministic per-mode seed), and $(\cdot)^{\leftarrow}$ is channel reversal — a coupling that makes the manifold curved, so the joint cross-channel structure is non-trivial (a fault can be marginally-valid yet jointly-invalid).

Imbalance (Zipf): mode frequencies $\pi_k\propto \mathrm{rank}(k)^{-\beta}$ are shuffled so identity is decoupled from frequency; the rarest modes get $O(1)$ samples. Bounded modes: the regime coordinate is a truncated/clipped Gaussian, $u\sim\mathcal N(0,\sigma_k^2 I_r)$ clipped to $\pm2\sigma_k$ — hard edges, empty between-mode regions. Heterogeneous sensing: correlated + heteroscedastic noise $\Sigma_\varepsilon=WW^\top+\operatorname{diag}(s^2)$, and per-channel quantization step $\delta_c$ (fine ADC-like to coarse near-binary). Standardisation uses train statistics only.

3. Static generator (make_miim)

3.1 Normal data

Draw mode counts $\sim\mathrm{Multinomial}(n,\pi)$; per mode, sample $u$, map through $g_k$, add correlated noise, quantize. The test-normal set is deliberately rare-enriched (half by the true prior, half forced from the rare tail) so the rare-mode false-positive rate is measurable.

3.2 Fault families (static)

  • OOD — points far outside every mode (uniform in a wide box, then pushed further from the nearest centroid).
  • pocket — a midpoint $x=(1-t)c_a+t\,c_b$, $t\in[0.35,0.65]$, between two well-sampled mode centroids: the over-interpolation false-negative case.

3.3 Non-circular labelling & distance

Labels never use the detector. A true-normal oracle holds a large fresh normal sample from the same modes/frame; a candidate is a genuine anomaly iff its $\kappa$-NN distance to that reference exceeds the $99^{\text{th}}$ percentile of normal self-distance (self-match dropped). A model-agnostic distance to the nearest mode uses a pooled within-mode Mahalanobis $\sigma(x)=\min_k\sqrt{(x-\bar x_k)^\top S^{-1}(x-\bar x_k)}$; the normal envelope is $E=Q_{0.99}(\sigma)$, so a fault at $\sigma(x)/E\in[1,2]$ is "just outside" (hard).

4. Temporal generator (make_temporal_miim)

Adds the axes the static one lacks — dynamics, transitions, dwell, temporal multiscale — so the history-dependent anomalies can be expressed and the trajectory channel validated.

4.1 Mode-transition state machine

Each mode has a small set of allowed successors (a sparse graph with Dirichlet-weighted edges); every other edge is forbidden. Per-mode dwell times are log-uniform $\in[15,220]$ (some regimes brief, some long — a temporal-scale mix). Stationary mode frequency is Zipf-imbalanced.

4.2 Within-mode dynamics & temporal multiscale

Inside a dwell the regime coordinate is an Ornstein-Uhlenbeck walk mean-reverting to the mode centre, clipped (bounded):

$$u_t=0.85\,u_{t-1}+0.15\,\varepsilon_t,\quad \varepsilon_t\sim\mathcal N(0,\sigma_k^2 I_r),\qquad u_t\leftarrow\operatorname{clip}(u_t,\pm2\sigma_k),$$

emitted through $g_k$. The stream is then smoothed per channel with different time constants $x^{(c)}_t=\alpha_c x^{(c)}_t+(1-\alpha_c)x^{(c)}_{t-1}$, $\alpha_c\in[0.05,0.95]$ — slow/inertial channels (temperature-like) vs fast (RPM-like): the temporal multiscale of assumption A9. Heterogeneous noise and quantization follow, then per-train standardisation and windowing (window $W$, stride $S$; per-window label by majority, plus the dominant anomaly type and mode).

4.3 Fault families (temporal) — snapshot vs history-dependent

  • snapshot pocket — a short between-two-modes segment.
  • snapshot drift — the OU coordinate slowly pushed out of bounds over a segment (bypasses the clip) → values leave the manifold.
  • history bad_transition — a forbidden mode→mode jump. Crucially the labelled windows are the ones settled inside the (valid) target mode, after the straddle: their values are indistinguishable from normal — only the path is invalid, so no snapshot detector can catch them; only the trajectory channel can.
Validation (snapshot detector, VaDE, per-type TPR @5% FPR). pocket 0.72, drift 0.91 (both snapshot-detectable) — but settled bad_transition 0.057 ≈ the false-positive floor: the snapshot channel is blind to the history-dependent anomaly, exactly the controlled case the real testbeds could not provide.

6. What the generators enable

  • ground-truth modes and anomaly types for per-type diagnostics;
  • a non-circular oracle and a model-agnostic distance/envelope for honest, detector-independent labels and difficulty bands;
  • a controlled history-dependent anomaly (bad_transition) that isolates the trajectory channel — the reason the temporal generator exists;
  • label-free hooks: C1-style generated anomalies for model selection, and the memory-horizon probe for B.
