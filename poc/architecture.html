Trajectory-aware anomaly detection: three models (A / B / C)

A frozen mode-encoder (A) and a queryable history encoder (B) provide context to a trajectory-conditional detector (C) that runs the C1–C4 components. Extends the static MIIM framework to time-series / dynamics.

Guiding principles. (1) Separation of concerns — three independent models (separate weights & training; A and C may share architecture, not weights). (2) Encode history, never forecast — the twin (A+B) only summarises the determined past, so unpredictable-but-normal signal is never treated as anomalous. (3) Condition, don't concatenate — context modulates the detector (FiLM / cross-attention) and is never a reconstruction target, so a normal history cannot dilute a real window anomaly. (4) One unified detector — with a faithful B, snapshot, contextual, and transition anomalies all surface as an off-manifold $(\text{window},c_t)$ pair, so detection is a single C-score (no explicit transition graph, no score-OR). (5) Normal-only training throughout.

Data flow: window → A → (γ, d); γ → B → c_t; (raw window, c_t) → C → anomaly score.

In plain terms. A single snapshot can't tell if you got here the wrong way. So three models split the work: A looks at one window and says which operating mode it is (a soft "90% mode 3, 10% mode 7") and how far it is from every mode. B watches those mode-readings over time and keeps a running, queryable memory of "which modes we've been in, in what order, for how long" — it only remembers, it never predicts or judges. C is the detector: it looks at the current window and is handed B's memory, then decides if anything is wrong. The memory is used to shape "what's normal here," not glued on as extra numbers, so a normal history can never hide a real problem.
Notation & naming. We standardise on the mixture-model terms and retire "assignment vector" (it implies a hard pick, whereas the quantity is soft).
$\gamma\in\Delta^K$responsibility vector (soft mode membership, $\gamma_k=P(k\mid z)$) — A's identity output, B's input.
$d\in\mathbb R^K$mode-distance vector (Mahalanobis to each component) — A's geometry output.
$\pi\in\Delta^K$mode prior (marginal responsibility mass, $\pi_k=\mathbb E[\gamma_k]$) — A's frequency output; drives B's rarity term and C's $\pi_k(c_t)$.
$e_t$step embedding (soft mixture of mode embeddings) — B's per-step input token.
$c_t$trajectory context (multi-resolution history readout) — B's output, C's conditioning.
Model short-names, used interchangeably with A/B/C: A = Mode Encoder, B = History Encoder, C = Conditional Detector.

Model A — mode discovery & geometry-aware window encoder

In plain terms
Looks at one window and reports two things: which operating mode it is (a soft "90% mode 3, 10% mode 7"), and how far it sits from every mode. The "which" is for the historian; the "how far" is for the detector (a window far from everything still gets a confident nearest-mode, so the assignment alone would hide it — the distances don't).
Architecture
VaDE: a VAE with a Gaussian-mixture latent prior of $K$ components (joint encode + cluster in latent space). Over-provisioned $K$; the anti-collapse recipe (variance floor, $\beta$ warm-up, covariance penalty, slow mixture-LR) keeps the imbalanced rare modes alive. No FiLM.
Input
a single window $\phi(W_t)$ (features or raw).
Output (per window)
latent $z$; responsibility vector $\gamma\in\Delta^K$ ($\gamma_k=P(\text{mode }k\mid z)$); mode-distance vector $d\in\mathbb R^K$, Mahalanobis in latent $d_k=(z-\mu_k)^\top\Sigma_k^{-1}(z-\mu_k)$; and, once over the training set, the mode prior $\pi_k=\mathbb E[\gamma_k]$ (the marginal frequency B and C need for rarity).
Loss
VaDE evidence lower bound (reconstruction + mixture-KL) + the anti-collapse terms. Normal data only.
Rationale
$\gamma$ is the regime identity (vocabulary for B); $d$ is the geometry — $\gamma$ is a softmax of $-d$ and loses magnitude (an OOD window still has a confident nearest mode), whereas $d$ preserves "how far off-manifold," which detection needs.
Objectives
recover the massive, imbalanced, implicit multimodality; provide a stable discrete vocabulary.
Constraints
normal-only; frozen after training so B's vocabulary can't drift; no anomaly logic; $K$ large with the imbalanced tail preserved.

Model B — trainable trajectory encoder (queryable history)

In plain terms
Keeps a running "diary" of which modes the system has been in and in what order. It only remembers the past — it never predicts the future (much normal behaviour is unpredictable) and never judges anomalies. We check it is a good diary by quizzing it: "which mode were we in k steps ago?" — if it answers far-back questions, it has kept the history.
Embedding (not tokenization)
A's modes are fuzzy (a soft $\gamma_t$), so hard tokenization is wrong. Embed each step as a soft mixture of learned mode embeddings,
$$e_t=\sum_k \gamma_{t,k}\,m_k,$$
so a between-modes window sits on the segment between $m_A$ and $m_B$: a transition is a smooth $e$-trajectory and a forbidden path is an unusual one. ($m_k$ init from A's centroids; a plain MLP on $\gamma_t$ is the simple equivalent.)
Encoder
one long-range sequence model over $\{e_t\}$ — SSM / Mamba (or dilated-TCN), streaming with $\sim\!10^3$-step memory. Its recurrent state $s_t$ is the memory; a readout of $s_t$ gives the context $c_t$.
Input
the stream of responsibilities $\gamma_t$ from frozen A.
Output
trajectory context $c_t$ (a vector, or a few memory tokens for C's cross-attention).
Loss — rare-weighted recall (no forecasting)
One head. Over a spread of offsets $k\in\{1,2,4,\dots,512\}$, recover the past mode from the context:
$$\mathcal L_B=\sum_t\sum_k \lambda_k\,\mathrm{CE}\big(\gamma_{t-k},\,r_\theta(c_t,\phi(k))\big),\qquad \lambda_k\propto 1/\pi.$$
Far offsets force long memory; asking only about the determined past keeps it unpredictability-proof; the rare-weight $\lambda_k\propto1/\pi$ stops the common mode from dominating, so brief rare visits are not forgotten. The learned selective state (Mamba's gate) does the rest — it writes more on novel/rare steps than on repeats, learned end-to-end from this loss.
Rationale
fuzzy modes → embed, don't tokenize; the past is determined so retrieval is well-posed; offset-parameterisation gives a tunable memory horizon. A faithful $c_t$ lets C catch invalid paths without an explicit transition graph.
Optional extensions (add only if the lean core misses something)
  • Containment head — "was mode $a$ present in period $P$?" (BCE, rare/common-balanced): coarse, timing-robust recall for long horizons, if precise offset-recall degrades too fast.
  • Explicit surprise-gated writing — if the learned gate under-writes rare states, force it: $s_t=s_{t-1}+g_t\Delta_t$ with $g_t=\sigma\big(\mathrm{MLP}(e_t,s_{t-1},-\log\pi_{k_t},\lVert e_t-s^{\text{recent}}\rVert)\big)$ (surprise = rarity $-\log\pi$ + change from recent; non-forecasting; debounce flicker).
  • Presence features — per mode "time since last visited", so a one-step rare visit stays on the record regardless of duration.
  • Multi-resolution + maturity — split $c_t$ into short/med/long readouts with a per-resolution maturity signal, if a single horizon is not enough.
Objectives
a faithful, queryable record of regime occupancy, order, and dwell — good enough that C catches invalid paths without an explicit transition graph.
Constraints
normal-only; reset state at run boundaries; no forecasting, no anomaly logic; frozen after training. Acceptance test: query-accuracy vs offset = the effective memory horizon.

Model C — trajectory-conditional anomaly detector

In plain terms
The actual detector. It looks at the current raw window and is told B's history, then asks "is this window normal given how we got here?" The history is used to tune expectations (not appended as extra numbers — that would let a normal history water down a real anomaly). One question — "is this (window, history) pair normal?" — catches all three fault kinds: a bad window on its own, a valid value that's wrong for the context, and a valid state reached by an impossible path. C1–C4 run on top unchanged.
Architecture
a VaDE (its own mixture prior, own weights) over the raw window, with the trajectory context injected by conditioning: FiLM $h'=\gamma(c_t)\odot h+\beta(c_t)$ when $c_t$ is a single vector, or cross-attention when B exposes a memory (window features query B's tokens). Same anti-collapse recipe as A. Optionally a $\pi_k(c_t)$ head so mode expectations are context-conditional.
Input
the raw window (local detail) + trajectory context $c_t$ via conditioning (never concatenated).
Output
a single anomaly score.
Loss
conditioned VaDE ELBO on normal + anti-collapse. The scoring components below are then fit on normal (and on C1's generated near-anomalies); none uses real fault labels.
Scoring — the C-score and its parts (C1–C4)
All parts are conditioned on $c_t$ and read on the window. Each targets a specific error the base score alone leaves open (E1 false-positive on a rare-but-valid mode; E2 false-negative on an over-interpolated pocket; E3 calibration without labels).
  • Base — whitened residual + latent NLL. Reconstruct the window, residual $r=x-\hat x$; energy $s_{\mathrm{rec}}=\tfrac12\,r^\top\hat\Sigma^{-1}r$ with $\hat\Sigma$ the Ledoit-Wolf shrunk residual covariance (so A7's correlated/heteroscedastic normal noise is discounted, not flagged), plus nearest-component latent NLL $s_{\mathrm{lat}}=\min_k d_k$. Why: off-manifold in value or in latent geometry.
  • C1 — physically-guided contrast (supplies the boundary). Manufactures hard, close pseudo-anomalies (channel-shuffle candidates kept only if off-mode yet physically plausible, difficulty set by the distance band $\sigma/E\in[1,2]$), context-conditional on $c_t$. Why: normal-only data has no hard negatives; C1 draws the decision surface exactly where E1/E2 live, and everything supervised (C3 member, C4 threshold) calibrates against it → addresses E3.
  • C2 — basin-of-attraction agreement. Descend the latent mixture potential $U(z)=-\log\sum_k\pi_k(c_t)\,\mathcal N(z;\mu_k,\Sigma_k)$ from $z$ to a fixed point $z^\star$; score the disagreement: path length $\lVert z-z^\star\rVert$ and whether $z^\star$'s basin matches the window's own responsibility. Normal windows sit in a basin and barely move; a pocket sits on a ridge and falls far into a basin it did not belong to. Why: catches the over-interpolation false-negative (E2) that a smooth decoder reconstructs too well.
  • C3 — ensemble (min-over-experts + supervised member). (a) One PCA/subspace expert per mode; the reconstruction score is the min over experts — a window is normal if some mode explains it, so a rare-but-valid mode is not penalised by the others. (b) A gradient-boosted member trained normal-vs-C1, class-balanced per mode. Why: a single global model over-smooths the imbalanced tail and false-positives on rare modes (E1); the per-mode min plus C1-supervised member sharpen the boundary.
  • C4 — mode-conditional risk control. Not one global threshold but a per-mode set; report the tail false-alarm rate $\mathrm{TFAR}=Q_{q}\big(\{\widehat{\mathrm{FPR}}_k:n_k\ge20\}\big)$ and its CVaR. Why: a global threshold is set by the common mode and silently sacrifices the rare modes' false-alarm (E1); TFAR makes the worst mode the objective, label-free (E3).
Fusion → one number. The C3 supervised member is the fuser: it consumes $[\,s_{\mathrm{rec}},\,s_{\mathrm{lat}},\,\text{C2 features},\,\text{per-mode PCA min}\,]$ and emits a single score, which C4 then calibrates per mode. So "one C-score" is literal, and every part is auditable.
Rationale
detect "normal given history." Conditioning (not concatenation) means $c_t$ shapes the expected-normal but is never reconstructed → no dilution. All anomaly families become an off-manifold $(\text{window},c_t)$ pair, so one detector suffices.
Objectives
a single unified detector covering snapshot, contextual, and transition anomalies, with per-mode false-alarm control.
Constraints
normal-only; reconstructs/scores only the window; $c_t$ modulates but is never a target; trained after A and B; different weights from A.

Detection & what each mechanism catches

Detection = the single C-score. With a faithful B, the explicit mode-transition graph is redundant (it is a hand-coded special case of C-conditioned-on-good-B) and is dropped.

Training pipeline

  1. Train A on normal (VaDE + anti-collapse) → freeze; emit $(\gamma,d)$ per window.
  2. Train B on A's $\gamma$-stream (queryable-past loss) → pass the memory-horizon test → freeze; emit $c_t$.
  3. Train C on normal, conditioned on $c_t$ (FiLM → cross-attention), fit C1–C4 on normal.
  4. Detect via the C-score.

Global objectives & constraints

  • Phenomena covered (validated on the temporal-MIIM benchmark): massive imbalanced multimodality; bounded modes; anomaly pockets; low-density (rare-valid) normals; heterogeneous + quantized sensing; temporal multiscale; a mode-transition state machine with forbidden edges; per-mode dwell.
  • Anomaly families: snapshot (pocket, drift), contextual, transition/path.
  • Invariants: normal-only training; encode history not forecast; condition not concatenate; single conditional detector; per-mode false-alarm control (C4/TFAR); label-free model selection (held-out normal loss, C1-generated anomalies, memory-horizon probe).

Design review

A candid pass over the recipe: the one risk that can break it, the parts that are probably more than needed, and the questions to settle before building.

Biggest risk — conditioning can explain away (the dual of dilution)

Principle (3) protects against a normal history diluting a real anomaly. The mirror risk is un-addressed: if FiLM / cross-attention is too expressive, a normal-looking history could let C reconstruct a genuinely off-manifold window and lower its score. The detection table's "snapshot → $c_t$ irrelevant" only holds if this cannot happen. Safeguard: residual FiLM initialised to identity ($\gamma\!\to\!1,\beta\!\to\!0$) and constrain context to raise, not fully suppress, the base off-manifold score (a non-negative context correction / monotone gate). Acceptance test: snapshot TPR must not fall when conditioning is switched on.

Redundancy / overkill (demote to fallback or gate behind an ablation)

  • B rare-state machinery (applied): the core now keeps just two mechanisms — rare-weighted recall + the learned selective state; the containment head, explicit surprise-gating, and presence features are demoted to optional extensions, added only if the core under-recalls rare states.
  • Resolutions (applied): the core reads out a single horizon; multi-resolution + per-resolution maturity is an optional extension, added only if one horizon proves too short.
  • C's $\pi_k(c_t)$ context-prior head overlaps with FiLM (both make expectations context-dependent). Ship FiLM alone first; add the head only if contextual/pocket faults survive.
  • C2 basin descent is the most expensive part (per-window latent optimisation) and only targets E2; the plain snapshot detector already reaches pocket $0.72$. Gate C2 behind an ablation showing it lifts pockets materially over base+C3.

Open questions to settle first

  • Run boundaries at inference: B resets state at run boundaries, but a streaming CPS may have no clean boundary — define the reset trigger, or a soft "session" state, else the maturity signal is ill-posed.
  • Non-stationarity / sensor drift vs A's frozen vocabulary (A1 assumes stationarity): per-run normalisation or a learned session regime?
  • Containment negatives must be plausible-but-absent (mirror C1's hard-negative principle), or the head learns a triviality.
  • Identifiability of B's mode embeddings $m_k$: initialise from A's centroids and regularise toward them, else free $m_k$ can drift or collapse.
  • Backbone for B (dilated-TCN vs SSM/Mamba) and its memory-horizon target; FiLM vs cross-attention cost/benefit; $c_t$ dimension vs the window; whether A and C share a frozen backbone + adapters.

Decisive experiment

Does C-alone lift bad_transition from the snapshot floor ($0.057$) toward the pocket/drift level on the temporal benchmark, with no transition graph? A yes confirms B is faithful and that principle (4) holds (the graph is redundant). This is the single result the whole three-model split is staked on.
